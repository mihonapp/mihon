# Roadmap

## Milestone 0: Repository foundation

- Yomori project identity and documentation
- Telemetry-free GitHub Actions APK builds
- Independent application ID and placeholder branding
- Upstream synchronization process

## Milestone 1: CBL core

- Safe CBL parser
- Domain model
- Fixtures and unit tests
- Reading-list persistence and migrations

## Milestone 2: Matching core

- Title and issue normalization
- Confidence scoring
- Ambiguity margin
- Persistent confirmed and rejected mappings

## Milestone 3: User workflow

- CBL import flow
- Extension selection and ranking
- Candidate review
- List, series, and entry overrides

## Milestone 4: Reading

- Cross-series next and previous navigation
- Reading-list progress
- Missing-entry handling
- Repair and rematching tools

## Milestone 5: Release readiness

- Final Yomori visual identity
- Release signing
- Backup and restore coverage
- Accessibility and physical-device testing
- Public release workflow and documentation
